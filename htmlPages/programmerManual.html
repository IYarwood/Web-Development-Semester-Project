On this page, you can find the technical overview of the website, summary of pages on this website, technical description of form elements, the decription of functions used on the back-end, and the Technical Overview of PHP Backend used for database.

Overview

This website allows students to register for courses through a simple web form. It also allows students to drop a course on a separate form. Users can also enroll students, add instructors, and add courses. In future updates, all of these changes will be saved to a database, allowing for easy updates to the college system. In addition, users will have the option to view who is registered for certain courses. All of these pages and functionalities can be seen listed above in the navigation bar, which will take you to whatever page you click. The forms on each page are of class "form-container" and the form elements within the group are of class "form-group". CSS files for each page type can be found in the "CSS" directory, and the html pages can be found in the "htmlPages" directory.

Function Descriptions

This project now uses a centralized JavaScript file validation.js that handles form validation for all pages.

validation.js

Event Listener
On DOM content load, it attaches a submit listener to the form with ID form. If the page is not in the ignore list (currently registerCourse.php or dropCourse.php), it calls validate_form().
validate_form(submitEvent)
Prevents form submission, collects all inputs under the .form-group class, and runs a matching validator from the inputObject map.
inputObject
A JavaScript object mapping input IDs (like coursePrefix, year, etc.) to validation functions. Only inputs with matching IDs are validated. Others are ignored.
Validation Functions
Each input field has a corresponding validation function, such as:
  • validate_firstname() – Checks non-empty string
  • validate_year() – Validates 4-digit year, must be 2025 or later
  • validate_course_prefix() – Must be 3–4 uppercase letters
  • validate_room_number() – Ensures format like "Selby 240"
  • validate_email() – Validates format abc123@marietta.edu
Fallback & Errors
If validation fails, an alert() is shown and submission is blocked. If any field doesn’t match the validation map or throws an error, it is caught and logged.
validate_none()
A dummy function used for fields that don’t need validation.

submitPopUp.js

get_form_data(form_container, days, time)
Parses the form by pairing labels and input/select values into a key-value map. For day/time fields, it uses a readable format (e.g., “Monday, Wednesday”).
format_alert_output(form_data)
Takes the map returned by get_form_data() and turns it into a multi-line alert message for confirmation before submission.
triggerPopUp(formElement, days, time)
Main exported function. It gets form data and shows a formatted alert() to the user with submitted information.

daysTimeDropdown.js

Purpose:
Dynamically shows time options in the dropdown based on the selected "Days Offered" value (MWF, MW, TTh, etc.).
showRelevantTimeDropdown()
Hides all time dropdowns initially, then reveals the appropriate group of times depending on the selected option.
Event Handling:
Runs on page load and when the user changes the selected day pattern.

viewCourse.js

Purpose:
Allows filtering course registrations based on student or instructor selection.
Key Behaviors:
  • Dropdown 1 filters options in Dropdown 2 (e.g., only students or only instructors)
  • Dropdown 2 filters the table view to show courses linked to the selected person
  • Hides unused rows dynamically using style.display = "none"

viewStudents.js

Purpose:
Displays a table of students based on the selected course or semester filter.
Course Filter:
When the user selects a course, shows students enrolled in that course using the data-courseid attribute.
Semester Filter:
When the user selects a semester, shows students enrolled in that semester using data-coursesemester.
UI Interaction:
Hides/show rows dynamically for better user experience when switching between filters.

Technical Overview of PHP Backend (Database Interaction)

1. studentEnroll.php

Purpose: Displays a form to enroll a new student. Collects student details and inserts them into the Students table in the database.

Process: HTML form submission using POST method to PHP backend. Captured fields are validated and inserted using SQL INSERT statements.

SQL Operations: INSERT INTO Students (firstName, lastName, year, major, email, status)

2. addInstructor.php

Purpose: Displays a form to add a new instructor. Collects instructor details and inserts into the Instructors table.

Process: Form collects first name, last name, department, rank, and email. Uses POST to send data to a PHP script that executes an INSERT query.

SQL Operations: INSERT INTO Instructors (firstName, lastName, department, rank, email)

3. addCourseSubmit.php

Purpose: Processes the course addition form and saves the course to the database.

Captured Fields: Semester, Year, Course Prefix, Number, Section, Name, Days, Time, Room, Credit Hours, Enrollment Cap, Instructor ID.

Process: The PHP script captures form data using POST and executes a parameterized SQL INSERT query.

SQL Operations: INSERT INTO Courses (semester, year, prefix, number, section, name, days, time, room, hours, cap, instructorID)

4. registerCourseSubmit.php

Purpose: Registers a student for a course by inserting a new registration record.

Process: - Captures student ID and course ID from form
- Performs SELECT to check if registration exists
- If not found, INSERT is executed to register the student

SQL Operations: SELECT * FROM Registration WHERE studentID = ? AND courseID = ?
followed by
INSERT INTO Registration (studentID, courseID) VALUES (?, ?)

5. dropCourseSubmit.php

Purpose: Drops a student’s enrollment from a course by deleting the registration record.

Process: Captures student ID and course ID from the form and deletes the corresponding entry from the Registration table.

SQL Operations: DELETE FROM Registration WHERE studentID = ? AND courseID = ?

General Notes for Developers

  • Database: MySQL using tables: Students, Instructors, Courses, Registration
  • Form Methods: POST (secure method)
  • Database Connection: mysqli_connect('localhost', 'root', '', 'database_name')
  • Security: All SQL queries now use prepared statements with mysqli_prepare() and bind_param(), which protects against SQL injection. Input is also sanitized with htmlspecialchars() to guard against cross-site scripting (XSS).
  • PHP Version: Compatible with PHP 7.x or higher (XAMPP recommended)
