Programmer Manual

On this page, you can find the technical overview of the website, summary of pages on this website, technical description of form elements, and the decription of functions used on the back-end.

Overview

This website allows students to register for courses through a simple web form. It also allows students to drop a course on a separate form. Users can also enroll students, add instructors, and add courses. In future updates, all of these changes will be saved to a database, allowing for easy updates to the college system. In addition, users will have the option to view who is registered for certain courses. All of these pages and functionalities can be seen listed above in the navigation bar, which will take you to whatever page you click. The forms on each page are of class "form-container" and the form elements within the group are of class "form-group". CSS files for each page type can be found in the "CSS" directory, and the html pages can be found in the "htmlPages" directory.

submitPopUp.js

on_load (deprecated)
Upon DOM content loading, get the whole form element and create an event listener for the form's submit button.
on_submit (deprecated)
Called by upon_load's event listener, on submit button press get form data using get_form_data and create the alert pop-up (using a string created by format_alert_output).
get_form_data
Called by triggerPopUp, search the form for label-input field pairs in each form-group in the form-container and map them to key/value pairs where the key is label's textContent attribute and the value is the input field's value attribute.
format_alert_output
Called by triggerPopUp, take the form information map created by get_form_data and use it to generate a string for the alert box. This string reads as a multi-line text box with each line as one key/value (field/value) pair and a title on the first line.
triggerPopUp
Called in addCourseValidation.js in order to trigger get_form_data only after all checks are done without error.
